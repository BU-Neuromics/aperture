# ADR-0016: Defer Bridge implementation; build and demo Aperture directly against Hippo

- **Status:** Accepted
- **Date:** 2026-06-15
- **Deciders:** labadorf, design session
- **Related:** ADR-0007, ADR-0008, ADR-0013; platform `sec6_security_model.md`; handoff §8 (sequencing)

## Context

Bridge — the auth gateway / PEP-PDP — is specced (platform `sec6_security_model.md`) but
unimplemented. Per sec6 §6.5, Aperture talks the **same autogenerated GraphQL contract**
whether it points at Hippo directly (local, no auth) or at Bridge (multi-user, enforcing); the
only variable is the injected capability-scoped client (ADR-0008). The Aperture interface is
higher-visibility and demonstrable sooner than an invisible auth gateway, so it carries a
bigger near-term payoff for showcasing the platform.

## Decision

**Defer Bridge *implementation*** and build/demonstrate Aperture directly against Hippo's
GraphQL endpoint. **The capability-scoped client *interface seam* (ADR-0008) is present from
day one** — locally a full-access / no-op pass-through. All data access (Aperture's *and*
components') routes through that abstraction now, so introducing Bridge later is a **swap of
the client implementation**, not a retrofit, with **no change to Aperture behavior or the
component contract**. Enforcement is never added to Aperture; it always remains Bridge's.

## Consequences

- Adding Bridge later changes one injected implementation, not call sites — preserving the
  ADR-0007 invariant ("safety universal from first keystroke") and avoiding a
  privilege-escalation retrofit across Aperture + components.
- Near-term deployments are single-user / local and **not access-controlled** — demos should
  use non-sensitive data or be explicit that they are a trust-the-network deployment.
- Field/slot-level masking arrives with Bridge; the view vocabulary and binding (ADR-0010,
  ADR-0015) must treat an absent/masked field as "not available," degrading gracefully
  (consistent with resolved-relationship fields returning `[]` today).
- Matches handoff §8 sequencing: schema-derived browse + faceted search first.

## Alternatives considered

- **Implement Bridge first.** Invisible plumbing; delays a demonstrable end-to-end portal and
  exercises Aperture later. Rejected for sequencing/payoff.
- **Defer Bridge *and* the capability-scoped client seam.** Would force retrofitting capability
  scoping across Aperture and every component later — the exact ADR-0007 violation. Rejected:
  the seam is cheap now and load-bearing.
