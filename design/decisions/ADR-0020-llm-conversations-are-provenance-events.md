# ADR-0020: LLM conversations and agent actions are provenance events

- **Status:** Accepted  — ⛔ **Deferred from MVP** (ADR-0026)
- **Date:** 2026-06-16
- **Deciders:** labadorf, design session
- **Related:** ADR-0018 (user-delegated authority), ADR-0019 (provider keys), ADR-0003/ADR-0017 (control plane), ADR-0021; handoff §6 (versioned, provenance-tagged, reversible writes); platform `sec6_security_model.md`

## Context

Agent-mediated changes must be auditable. Handoff §6 already requires config/registry writes
to be versioned, PROV-O-tagged, and reversible. Beyond the resulting write, we need to track
**the agent interaction that produced it** — and the requirement is explicit: **conversations
with LLMs are themselves provenance events.** We also need **observability** over all user/
agent interactions (usage, cost, outcomes) even though the LLM endpoint enforces its own
budgets — Aperture still owns attribution and visibility.

## Decision

Treat an LLM interaction as a **first-class, PROV-O-tracked control-plane entity**
(LinkML-on-Hippo; ADR-0017) linking:

> **user** (actor, ADR-0018) → **model + provider-key-ref** (ADR-0019) → the **tool calls /
> actions** issued → the **config revisions / queries** they produced (`wasGeneratedBy` /
> `used`) → **token usage**.

This yields an unbroken chain: *this config revision was generated by this agent turn, in this
conversation, initiated by this user, on this key, costing N tokens.* Per-user/per-agent
**observability** (token usage, action counts, latency, success/failure) layers on top.

**Scope by surface:**
- **External coding agent (MCP/API), now:** Aperture sees **authenticated tool calls**, not
  the chat transcript — record **action-level** provenance (each tool invocation as an
  attributed event, tagged with a conversation/session id when the agent supplies one).
- **In-app chat (deferred, ADR-0021):** Aperture hosts the loop and sees the **full
  transcript** — record the complete conversation when that surface is built.

## Consequences

- Agent edits are distinguishable from direct human edits in provenance (actor = user;
  mediation = agent) — satisfying ADR-0018's audit requirement.
- Aperture's control plane gains a conversation/interaction entity in its LinkML model.
- MCP tool calls should carry/attach a conversation or session id where available, so
  action-level events can be grouped into a turn.
- Foundation for cost attribution and abuse/anomaly detection per user and per key.

## Alternatives considered

- **Track only the resulting config revision.** Loses the *why/how* and the cost; can't tell
  an agent edit from a human one. Rejected.
- **Log conversations outside provenance** (separate app log). Breaks the single audit chain
  the platform relies on (handoff §6; the Bridge↔Hippo `request_id` correlation pattern).
  Rejected.
